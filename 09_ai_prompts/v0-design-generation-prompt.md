# v0 画面生成プロンプト

## v0にそのまま貼る完全版（外部ファイル参照なし）

```text
あなたは、SaaS管理画面に強いプロダクトデザイナー兼フロントエンド実装者です。
Instagram運用管理ツールのUIを、Next.js + React + TypeScript + Tailwind CSSで実装してください。

目的:
- 投稿コンテンツ管理と予約投稿を効率化する
- 投稿失敗時の復旧を素早く行える
- 運用KPI（投稿実行率、投稿頻度）を可視化する

今回実装する画面:
1. S2 ダッシュボード画面
2. S3 コンテンツ一覧画面
3. S4 コンテンツ編集画面

前提仕様（このプロンプト内で完結）:
- 投稿種別は image_feed / video_feed / carousel / reel をサポート
- コンテンツ状態は draft / scheduled / published / failed / action_required
- KPIは 投稿実行率、週次投稿本数、失敗件数、要対応件数
- 予約投稿は未来時刻のみ登録可能
- エラー種別は temporary_error（一時障害）と permanent_error（恒久障害）

画面要件:

[S2 ダッシュボード]
- KPIカード4つ（投稿実行率、週次投稿本数、失敗件数、要対応件数）
- 日次の予約数と実行数を並べたチャート
- 直近失敗ジョブ一覧（投稿名、失敗理由、発生時刻、再実行ボタン）
- 主要アクションボタン: 「新規投稿作成」

[S3 コンテンツ一覧]
- 検索バー
- フィルター（状態、投稿種別、期間）
- テーブル列: 投稿名 / 投稿種別 / 状態 / 予約日時 / 最終更新者 / アクション
- 行クリックで編集画面へ遷移できる導線（モックで可）
- アクション: 編集、複製、予約

[S4 コンテンツ編集]
- 投稿種別セレクト（image_feed / video_feed / carousel / reel）
- メディアアップロードエリア（複数ファイル対応UI）
- キャプション入力（最大2200文字、文字数カウンタ表示）
- ハッシュタグ入力
- バリデーション結果パネル
- ボタン: 「下書きを保存」「予約する」

状態設計（全画面共通）:
- 初期表示
- ローディング
- 空状態
- エラー状態
- 成功通知（トースト）

表示するエラー例:
- 「未入力の必須項目があります。入力してから再度お試しください。」
- 「公開日時は現在より後の時刻を指定してください。」
- 「この投稿種別では利用できないメディア形式です。」
- 「アカウント連携の有効期限が切れています。再連携してください。」

表示する通知例:
- 「予約を登録しました。指定時刻に自動投稿されます。」
- 「投稿が完了しました。」
- 「自動再試行中です（{retryCount}/{maxRetry}）。」

アクセシビリティ要件:
- WCAG 2.1 AA相当
- キーボードのみで主要操作が完結できる
- フォーカスリングを明確に表示
- 入力エラー文言をaria-describedbyで関連付け
- 状態表示は色だけに依存しない（アイコンやラベル併用）

レスポンシブ要件:
- mobile / tablet / desktop に対応
- テーブルは狭い画面でカード表示に切り替えてもよい

実装方針:
- App Router前提
- 必要な箇所のみ "use client" を使用
- 再利用コンポーネントに分割（KpiCard, StatusBadge, ContentTable, ValidationPanelなど）
- モックデータを同梱

出力形式:
1. ディレクトリ構成
2. ページ実装コード
3. 子コンポーネント実装コード
4. モックデータ
5. 要件対応表（どの要件をどこで満たしたか）
```

## 基本ルール

v0 を使って画面実装を生成する場合は、以下のテンプレートを使用してください。
本プロジェクトでは、Instagram運用（コンテンツ管理・予約投稿・障害復旧）を中心に、実装可能なUIを生成することを目的とします。

## プロンプトテンプレート

```text
あなたはプロダクトデザイナー兼フロントエンド実装者です。
以下の要件に従って、Next.js + React + TypeScript + Tailwind CSS のUIを生成してください。

目的:
- [画面の目的]

関連ドキュメント:
- 画面一覧: ../02_design/screen-list.md
- 画面仕様: ../02_design/screen-specification.md
- ユーザーフロー: ../02_design/user-flow.md
- UIコピー: ../02_design/ui-copy.md
- アクセシビリティ: ../02_design/accessibility-check.md
- 機能要件: ../01_requirements/functional-requirements.md

対象画面:
- [Sx 画面名]

必須UI要素:
- [要素1]
- [要素2]
- [要素3]

状態設計:
- 初期状態
- ローディング状態
- 空状態
- エラー状態
- 成功状態

インタラクション:
- [操作A] -> [期待動作]
- [操作B] -> [期待動作]

データモデル（モック可）:
- [項目名: 型]

実装制約:
- App Router 構成を前提
- クライアントコンポーネントが必要な箇所のみ "use client" を使う
- 再利用可能なUIコンポーネントに分割する
- レスポンシブ対応（mobile / tablet / desktop）
- WCAG 2.1 AA を満たす（キーボード操作、aria属性、十分なコントラスト）
- 文字列は日本語で実装する

デザイン方針:
- 運用ツールとして視認性と情報密度を重視
- KPIカード、テーブル、ステータスバッジを明確に差別化
- 状態色は色だけでなくアイコン/ラベルでも判別可能にする

出力形式:
1. ページ全体コード
2. 必要な子コンポーネントコード
3. モックデータ
4. 実装メモ（どの要件を満たしたか）
```

## そのまま使える v0 指示文（初版）

```text
Instagram運用管理ツールの管理画面を生成してください。
Next.js + React + TypeScript + Tailwind CSS を前提に、まずは以下3画面を実装してください。

対象画面:
1) S2 ダッシュボード画面
2) S3 コンテンツ一覧画面
3) S4 コンテンツ編集画面

機能要件の要点:
- 投稿実行率、週次投稿本数、失敗件数、要対応件数を可視化
- 下書き / 予約済み / 要対応 / 公開済みを一覧管理
- 投稿種別（画像フィード、動画フィード、カルーセル、リール）を切り替えて編集
- メディアバリデーション結果を明示
- 予約導線（「予約する」ボタン）を設置

画面ごとの要件:
- S2 ダッシュボード:
  - KPIカード4つ
  - 日次の予約数/実行数チャート
  - 直近失敗ジョブ一覧
  - 「新規投稿作成」ボタン
- S3 コンテンツ一覧:
  - 検索バー
  - ステータス、投稿種別、期間フィルター
  - テーブル列: 投稿名、種別、状態、予約日時、最終更新者、アクション
  - 行クリックで編集遷移（ダミーで可）
- S4 コンテンツ編集:
  - 投稿種別セレクト
  - メディアアップロードエリア（複数対応UI）
  - キャプション入力（最大2200文字カウンタ表示）
  - ハッシュタグ入力
  - バリデーション結果パネル
  - 「下書きを保存」「予約する」ボタン

状態設計:
- ローディング、空状態、エラー状態、成功トーストを含める
- エラー例: 必須項目未入力、メディア形式不正

アクセシビリティ要件:
- キーボード操作可能
- フォーカスリング明確
- 入力エラー文言をaria-describedbyで関連付け
- 色だけに依存しない状態表示

UIコピー（日本語）:
- ボタン: 「下書きを保存」「この日時で予約する」「再実行する」
- エラー: 「公開日時は現在より後の時刻を指定してください。」
- 通知: 「予約を登録しました。指定時刻に自動投稿されます。」

実装指示:
- コンポーネントを分割し再利用しやすくする
- モックデータを同梱する
- Tailwindクラスは可読性を重視
- 生成後に、各画面がどの要件を満たしているか箇条書きで示す
```
