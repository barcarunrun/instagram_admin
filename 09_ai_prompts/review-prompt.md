# コードレビュープロンプト

## 基本ルール

AI を使用してコードレビューする場合は、以下のプロンプトテンプレートを使用してください。

## プロンプトテンプレート

```
レビュー対象:
[レビュー対象コードのファイルパス]

関連ドキュメント:
- 要件: [01_requirements/...]
- API設計: [03_architecture/...]
- アーキテクチャ: [03_architecture/...]

レビュー観点:
- 機能要件を満たしているか
- セキュリティに問題がないか
- パフォーマンスに問題がないか
- テストカバレッジは十分か
- プロジェクトのコーディング規約に従っているか

指摘形式:
1. 問題点: [具体的な問題]
2. 重要度: [Critical / High / Medium / Low]
3. 改善案: [改善方法]
```

## 使用例

### 例1: API エンドポイントレビュー

```
レビュー対象: 05_source_code/backend/routes/users.js

関連ドキュメント:
- API設計: 03_architecture/api-design.md
- セキュリティ: 03_architecture/security.md

レビュー観点:
- 入力値検証
- エラーハンドリング
- 認証・認可
- SQL インジェクション対策
- レート制限
```

### 例2: React コンポーネントレビュー

```
レビュー対象: 05_source_code/frontend/components/UserProfile.jsx

関連ドキュメント:
- 画面仕様: 02_design/screen-specification.md
- アーキテクチャ: 03_architecture/architecture.md

レビュー観点:
- Props の型定義
- State 管理
- メモリリーク対策
- アクセシビリティ
- パフォーマンス (不要なレンダリング)
```

## レビュー結果の記録

レビュー後は、以下の形式で記録:

```
## コードレビュー完了

ファイル: [ファイルパス]
レビュアー: [レビュアー名]
日時: [日時]

### 指摘事項

1. [指摘内容]
   - 重要度: High
   - 修正方法: [修正案]

2. [指摘内容]
   - 重要度: Medium
   - 修正方法: [修正案]

### 総合評価

- 要件準拠: ✅ 合格
- セキュリティ: ⚠️ 要改善
- パフォーマンス: ✅ 合格
- テスト: ✅ 合格

### 承認

[ ] 修正不要で承認
[ ] 修正後に再レビュー
```
