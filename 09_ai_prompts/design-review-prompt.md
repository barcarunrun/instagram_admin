# デザインレビュープロンプト

## 基本ルール

AI を使用してデザインをレビューする場合は、以下のプロンプトテンプレートを使用してください。

## プロンプトテンプレート

```
レビュー対象:
[Figma リンク、または画面キャプチャのパス]

関連ドキュメント:
- 要件: [01_requirements/...]
- 画面仕様: [02_design/screen-specification.md](../02_design/screen-specification.md)
- アクセシビリティ: [02_design/accessibility-check.md](../02_design/accessibility-check.md)

レビュー観点:
- ユーザビリティ: 直感的に操作できるか
- 一貫性: 全体的なデザイン一貫性があるか
- アクセシビリティ: WCAG 2.1 準拠しているか
- 反応性: レスポンシブデザインか
- ブランド統一性: ブランドガイドラインに従っているか

フィードバック形式:
1. 良い点: [良い点]
2. 改善点: [改善点]
3. 疑問点: [疑問点]
```

## 使用例

### 例1: ユーザー登録画面レビュー

```
レビュー対象:
Figma: [リンク]

関連ドキュメント:
- 画面仕様: 02_design/screen-specification.md (画面1)
- ユーザーストーリー: 01_requirements/user-stories.md

レビュー観点:
- フォーム のレイアウト: 入力フィールド の配置は最適か?
- バリデーション表示: エラーメッセージの表示位置は適切か?
- 操作性: キーボード操作のみで完結できるか?
- モバイル対応: スマートフォンで使いやすいか?
```

### 例2: 投稿フィード画面レビュー

```
レビュー対象:
05_source_code/frontend/screens/Feed.jsx (スクリーンショット)

関連ドキュメント:
- 画面仕様: 02_design/screen-specification.md (画面2)
- UI コピー: 02_design/ui-copy.md

レビュー観点:
- 情報階層: 最も重要な情報が目立っているか?
- スペーシング: 適切な余白があるか?
- 配色: コントラスト比は十分か?
- インタラクション: クリック対象が明確か?
```

## デザインレビューチェックリスト

- [ ] 画面仕様を満たしている
- [ ] ユーザーストーリーを支援する設計
- [ ] WCAG 2.1 AA レベル準拠
- [ ] モバイル・タブレット・デスクトップ対応
- [ ] ダークモード対応 (必要に応じて)
- [ ] アニメーション/トランジションが過度でない
- [ ] ローディング状態が表示される
- [ ] エラー状態が適切に表示される
- [ ] 成功状態がユーザーにわかりやすい
