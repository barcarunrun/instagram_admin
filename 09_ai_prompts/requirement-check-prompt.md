# 要件確認プロンプト

## 基本ルール

AI を使用して要件を確認する場合は、以下のプロンプトテンプレートを使用してください。

## プロンプトテンプレート

```
確認対象:
[確認する仕様書、ドキュメントなど]

確認項目:
- 機能の完全性: すべての機能が定義されているか
- 受入基準の明確性: 受け入れ判定が客観的か
- テストカバレッジ: すべての機能にテストケースがあるか
- エッジケース: エッジケースが定義されているか
- 非機能要件: パフォーマンス、セキュリティ要件が定義されているか
- 実装可能性: 技術的に実装可能か

確認観点:
- [観点1]
- [観点2]

質問事項:
- [質問1]
- [質問2]
```

## 使用例

### 例1: PRD 確認

```
確認対象:
01_requirements/prd.md

確認項目:
- 機能の完全性: 全機能が定義されているか
- スコープの明確性: スコープインとスコープアウトが明確か
- 優先度: 優先度が設定されているか
- 制約条件: 技術的制約が記載されているか

質問事項:
- ユーザー登録機能はいつまでに実装する予定か?
- API レート制限はどの程度の値を想定しているか?
```

### 例2: ユーザーストーリー確認

```
確認対象:
01_requirements/user-stories.md

確認項目:
- 受入基準の明確性: 受け入れ判定が客観的か
- テストケース: すべてのストーリーにテストケースがあるか
- 依存関係: ストーリー間の依存関係が明確か
- 実装可能性: 1 スプリント (2 週間) で実装可能か

質問事項:
- 「ユーザーがフォロー機能を使える」の実装見積は?
```

## 確認結果の記録

確認後は、以下の形式で記録:

```
## 要件確認完了

確認対象: [ドキュメント名]
確認者: [確認者名]
日時: [日時]

### 確認結果

#### ✅ 問題なし
- [項目名]

#### ⚠️ 要修正
- [項目名]
  - 問題: [具体的な問題]
  - 修正案: [修正案]

#### ❓ 要確認
- [項目名]
  - 質問: [質問内容]
  - 回答: [回答]

### 総合評価

- 完全性: [80%/100%]
- 明確性: [90%/100%]
- 実装可能性: [✅ 可能 / ⚠️ 要検討]

### 次のアクション

[ ] ドキュメント修正が必要
[ ] PdM との確認が必要
[ ] 承認可能
```
