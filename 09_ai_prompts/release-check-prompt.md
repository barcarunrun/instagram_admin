# リリースチェックプロンプト

## 基本ルール

AI を使用してリリース前のチェックを行う場合は、以下のプロンプトテンプレートを使用してください。

## プロンプトテンプレート

```
リリース対象:
[バージョン番号、リリース日時]

確認対象:
- コード: [提供するコードまたはプルリクエスト番号]
- テスト結果: [テスト結果の要約]
- ドキュメント: [更新されたドキュメント]

確認項目:
- 要件充足: すべての要件が実装されたか
- テスト: テストカバレッジは十分か
- セキュリティ: セキュリティテストは完了したか
- パフォーマンス: パフォーマンステストは完了したか
- ドキュメント: ドキュメントは最新か
- リリースノート: リリースノートは準備されているか
- ロールバック計画: ロールバック計画は準備されているか
- デプロイメント計画: デプロイメント計画は準備されているか

チェック観点:
- [観点1]
- [観点2]

質問事項:
- [質問1]
- [質問2]
```

## 使用例

### 例1: Version 1.0.0 リリース前チェック

```
リリース対象:
Version 1.0.0 (2024-01-15)

確認対象:
- コード: PR #123, #124, #125
- テスト結果: ユニット 85%, 統合 100%, E2E 95%
- ドキュメント: README, API Doc, Release Notes

確認項目:
- 要件充足: PRD のすべての要件が実装されたか?
- テスト: テストカバレッジ 80% 以上か?
- セキュリティ: OWASP Top 10 対策は実施されたか?
- バグ: Critical, High バグはないか?
- ドキュメント: リリースノート、マイグレーション手順は準備されているか?

チェック観点:
- ユーザー登録機能: 正常に動作するか?
- 投稿機能: 正常に動作するか?
- パフォーマンス: ページロード時間は目標値以内か?
```

## リリースチェックチェックリスト

- [ ] すべての機能要件が実装されている
- [ ] テストカバレッジが目標値を達成
- [ ] Critical, High バグがない
- [ ] セキュリティテストが完了
- [ ] パフォーマンステストが完了
- [ ] リリースノートが準備されている
- [ ] API ドキュメントが最新
- [ ] 環境構築手順が最新
- [ ] マイグレーション計画が準備されている
- [ ] ロールバック計画が準備されている
- [ ] デプロイメント計画が準備されている
- [ ] 利害関係者の承認が完了している

## リリース承認基準

- [ ] 要件充足度: 100%
- [ ] テストカバレッジ: 80%以上
- [ ] バグ: Critical 0, High 0
- [ ] ドキュメント完全性: 100%
- [ ] セキュリティ: 問題なし
- [ ] パフォーマンス: 目標値を達成
