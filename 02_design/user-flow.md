# ユーザーフロー

## 主要フロー

### フロー 1: 初回連携から予約投稿まで

```
[開始]
-> [S1 連携実行]
-> [S2 ダッシュボード確認]
-> [S4 コンテンツ作成]
-> [S5 予約登録]
-> [S6 カレンダー確認]
-> [終了]
```

**説明:** 新規利用者がアカウント連携を完了し、最初の予約投稿を成立させる標準導線。

### フロー 2: 下書き管理から複数投稿を計画

```
[開始]
-> [S3 コンテンツ一覧で既存下書きを確認]
-> [S4 コンテンツ複製・編集]
-> [S5 複数日程を予約]
-> [S6 週次表示で密度調整]
-> [終了]
```

**説明:** 既存資産を再利用し、週次で計画投稿を組む運用者向け導線。

### フロー 3: 失敗投稿の復旧

```
[開始]
-> [S2 アラートで失敗を検知]
-> [S7 実行ログで原因確認]
-> [S4 コンテンツ修正 または S1 再認可]
-> [S7 再実行]
-> [S6 結果確認]
-> [終了]
```

**説明:** 投稿失敗時に原因を特定して再実行し、投稿実行率を維持する導線。

## オルタナティブフロー

### 代替フロー A: トークン期限切れ

```
[S5 実行時]
-> [認証エラー検出]
-> [S1 再認可誘導]
-> [S7 再実行]
```

### 代替フロー B: 予約時刻不正（過去日時）

```
[S5 予約登録]
-> [バリデーションエラー表示]
-> [日時再入力]
-> [再登録]
```

### 代替フロー C: レート制限

```
[S5/S7 投稿実行]
-> [一時エラー分類]
-> [自動再試行]
-> [成功 または 要対応化]
```
