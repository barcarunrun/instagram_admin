# アクセシビリティチェック

## WCAG準拠

- WCAG 2.1 Level AA に準拠

## チェックリスト

- [ ] 状態表示（成功/失敗/要対応）を色以外のテキストやアイコンでも伝える
- [ ] 本文サイズ相当のテキストはコントラスト比4.5:1以上
- [ ] 主要操作（作成、保存、予約、再実行）がキーボードのみで完結できる
- [ ] モーダル（S5）表示時にフォーカストラップされ、閉じると元要素に戻る
- [ ] 画像プレビュー、ログテーブル、カレンダーイベントに適切な代替テキストを付与
- [ ] 入力エラー時にエラー文言をフィールドに関連付けて読み上げ可能
- [ ] フォーカスインジケータは常時視認可能（非表示にしない）
- [ ] トグル、ボタン、リンクに操作目的を示すラベルを付与

## テスト方法

1. 画面単位の静的検査
	axe DevToolsとLighthouseで各画面（S1-S8）を検査する。
2. キーボード操作検証
	Tab/Shift+Tab/Enter/Escapeのみで主要導線を操作し、詰まりがないことを確認する。
3. スクリーンリーダー検証
	VoiceOverでフォーム入力、エラー表示、モーダル操作、カレンダーイベント読み上げを確認する。
4. コントラスト検証
	デザインカラーの組み合わせをWAVEまたはコントラストチェッカーで計測する。

## ツール

- WAVE
- axe DevTools
- Lighthouse
- VoiceOver (macOS)
