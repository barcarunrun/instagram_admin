# 画面仕様

## 画面 S1: ログイン / アカウント連携画面

### 概要

Facebook OAuthとInstagramアカウント連携を実施し、投稿可能状態を作る。

### レイアウト

- ヘッダー: サービス名、ログイン状態
- メイン: 連携ステップ（認可 -> アカウント選択 -> 権限確認）
- サイド: 連携状態カード（有効 / 期限切れ / エラー）

### ユーザーインタラクション

- 「Facebookで連携する」: OAuth画面へ遷移する
- 「このアカウントで接続」: 選択アカウントで連携を確定する
- 「再認可する」: 期限切れ状態から再連携フローを再開する

### 入力フィールド

| フィールド | 型 | 必須 | 検証 |
|--|--|--|--|
| Facebookログイン | OAuth | Yes | 認可成功時のみ次へ進む |
| Instagramアカウント選択 | Select | Yes | 投稿権限がある候補のみ表示 |
| Facebookページ選択 | Select | Yes | 選択アカウントと紐づくページのみ選択可 |

### API呼び出し

- GET /api/auth/instagram/oauth-url
- GET /api/auth/instagram/callback
- POST /api/integrations/instagram/connect
- GET /api/integrations/instagram/status

## 画面 S2: ダッシュボード画面

### 概要

投稿運用の現状を把握するためのホーム画面。KPIカードとアラートを表示する。

### レイアウト

- KPIカード: 投稿実行率、週次投稿本数、失敗件数、要対応件数
- チャート: 日次の予約数と実行数
- 一覧: 直近失敗ジョブ、要再認可アカウント

### ユーザーインタラクション

- KPIカード押下: 該当する詳細画面へ遷移する
- 失敗ジョブ押下: S7へ遷移する
- 「新規投稿作成」押下: S4へ遷移する

### 入力フィールド

| フィールド | 型 | 必須 | 検証 |
|--|--|--|--|
| 期間フィルター | DateRange | No | 開始日 <= 終了日 |
| アカウントフィルター | Select | No | 連携有効アカウントのみ |

### API呼び出し

- GET /api/dashboard/kpi
- GET /api/dashboard/alerts

## 画面 S3: コンテンツ一覧画面

### 概要

下書き、予約済み、要対応投稿を横断管理する。

### レイアウト

- 検索バー: キーワード検索
- フィルター: ステータス、投稿種別、期間、アカウント
- テーブル: 投稿名、種別、状態、予約日時、最終更新者

### ユーザーインタラクション

- 「新規作成」: S4へ遷移する
- 行選択: S4（編集）へ遷移する
- 「複製」: 同内容で新規下書きを作成する

### 入力フィールド

| フィールド | 型 | 必須 | 検証 |
|--|--|--|--|
| 検索キーワード | Text | No | 最大100文字 |
| ステータス | MultiSelect | No | draft, scheduled, failed, published |
| 投稿種別 | MultiSelect | No | image, video, carousel, reel |

### API呼び出し

- GET /api/contents
- POST /api/contents/{id}/duplicate

## 画面 S4: コンテンツ編集画面

### 概要

投稿種別に応じてメディア、キャプション、タグを編集し下書き保存する。

### レイアウト

- 左: メディアアップロードとプレビュー
- 右: 種別設定、キャプション、ハッシュタグ、バリデーション結果
- 下部: 保存、承認申請、予約設定ボタン

### ユーザーインタラクション

- 投稿種別変更: 必須項目UIを切り替える
- 下書き保存: バリデーション後に保存する
- 「予約する」: S5を開く

### 入力フィールド

| フィールド | 型 | 必須 | 検証 |
|--|--|--|--|
| 投稿種別 | Select | Yes | image/video/carousel/reel/extension |
| メディア | File[] | Yes | 種別別の形式・サイズ・尺 |
| キャプション | TextArea | Yes | 最大2200文字 |
| ハッシュタグ | TextArea | No | 1件30文字以内、最大30件 |

### API呼び出し

- POST /api/contents
- PUT /api/contents/{id}
- POST /api/contents/{id}/validate

## 画面 S5: 予約設定モーダル

### 概要

公開日時と公開先を指定して投稿予約を登録する。

### レイアウト

- 日時入力
- タイムゾーン選択
- 公開先アカウント選択
- 予約可否チェック結果

### ユーザーインタラクション

- 「予約登録」: 妥当性確認後にジョブを作成する
- 「キャンセル」: モーダルを閉じる

### 入力フィールド

| フィールド | 型 | 必須 | 検証 |
|--|--|--|--|
| 公開日時 | DateTime | Yes | 現在時刻より未来 |
| タイムゾーン | Select | Yes | IANA TZ形式 |
| 公開先アカウント | Select | Yes | 連携有効アカウントのみ |

### API呼び出し

- POST /api/schedules
- POST /api/schedules/validate

## 画面 S6: 投稿カレンダー画面

### 概要

予約投稿と実績投稿を日/週/月ビューで確認する。

### レイアウト

- ビュー切替（Day/Week/Month）
- カレンダー本体（状態別色分け）
- サイド詳細（選択イベントの詳細情報）

### ユーザーインタラクション

- イベント押下: 詳細と実行ログを表示
- 日付押下: 新規予約の初期日時としてS5を開く

### 入力フィールド

| フィールド | 型 | 必須 | 検証 |
|--|--|--|--|
| 表示期間 | DateRange | Yes | 最大3か月表示 |
| 表示対象 | MultiSelect | No | scheduled/published/failed |

### API呼び出し

- GET /api/calendar/events

## 画面 S7: 投稿実行ログ画面

### 概要

投稿ジョブの成功/失敗履歴と再試行結果を確認し、手動再実行する。

### レイアウト

- 一覧: 実行時刻、状態、エラー分類、再試行回数
- 詳細パネル: エラー内容、外部レスポンス、対処ガイド

### ユーザーインタラクション

- ログ行押下: 詳細表示
- 「再実行」: 手動再実行を実行する

### 入力フィールド

| フィールド | 型 | 必須 | 検証 |
|--|--|--|--|
| 状態フィルター | MultiSelect | No | success, failed, retrying, action_required |
| 期間フィルター | DateRange | No | 開始日 <= 終了日 |

### API呼び出し

- GET /api/jobs/logs
- POST /api/jobs/{jobId}/retry

## 画面 S8: 通知 / 監査ログ設定画面

### 概要

通知チャネル設定と監査ログ参照条件を管理する。

### レイアウト

- 通知設定セクション
- 監査ログ検索セクション
- 変更履歴セクション

### ユーザーインタラクション

- 通知ON/OFF切替: 即時保存
- 「テスト通知送信」: 通知到達確認

### 入力フィールド

| フィールド | 型 | 必須 | 検証 |
|--|--|--|--|
| メール通知有効 | Toggle | No | Boolean |
| チャットWebhook URL | URL | No | HTTPS必須 |
| ログ検索キーワード | Text | No | 最大100文字 |

### API呼び出し

- GET /api/notifications/settings
- PUT /api/notifications/settings
- GET /api/audit-logs
