# ADR-004: UI デザイン方針

## ステータス

提案中

## 背景

ユーザーエクスペリエンスは、プロダクト成功の重要な要因である。

一貫性のあり、使いやすい UI デザイン方針を定める必要がある。

## 決定

### デザインシステム

- **Design Token ベース**: Figma での管理
- **コンポーネントライブラリ**: React Component Library
- **カラーパレット**: カスタムカラー定義

### カラースキーム

- **プライマリーカラー**: #007AFF (Instagram Blue)
- **セカンダリーカラー**: #E1306C (Instagram Pink)
- **グレースケール**: #000, #333, #666, #999, #CCC, #EEE, #FFF

### タイポグラフィ

- **フォント**: Inter, Segoe UI など
- **フォントサイズ**: 12px, 14px, 16px, 18px, 20px, 24px, 32px

### スペーシング

- **Base Unit**: 8px
- **マージン・パディング**: 8px, 16px, 24px, 32px 単位

### アイコン

- **ライブラリ**: Heroicons, Material Icons など
- **サイズ**: 16px, 20px, 24px, 32px

## 理由

### Design Token による利点

- デザイン変更が容易
- 設計と開発の連携が効率化
- ダークモード対応が容易

### カラースキーム

- Instagram ブランドとの統一性
- 視認性と親しみやすさ

## 代替案

### 代替案 1: Atomic Design

- **利点**: コンポーネント設計が明確
- **欠点**: 学習コストが高い

### 代替案 2: Material Design

- **利点**: Google の統一規格
- **欠点**: Instagram ブランドとの統一性が低い

## 実装ガイドライン

### コンポーネント開発

1. Design Token を使用
2. 小さく再利用可能なコンポーネントから構築
3. Props を明確に定義
4. Storybook での記録

### ダークモード対応

- CSS カスタムプロパティで切り替え
- `prefers-color-scheme` メディアクエリ対応

## 影響

### ポジティブな影響

- 一貫性のある UI
- 開発速度が向上
- ダークモード対応が容易

### ネガティブな影響

- 初期実装コストが高い
- 設計ルール遵守の徹底が必要
