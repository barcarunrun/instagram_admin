# セキュリティ

## 認証・認可

### 認証方式

- 管理画面利用者の認証はアプリケーション側セッション認証を利用する。
- Instagram連携はFacebook OAuth 2.0を利用する。
- 内部API通信は短期トークンまたはサービスシークレットで保護する。

### パスワードポリシー

- アプリ独自ログインを持つ場合は12文字以上、英字・数字・記号の組み合わせを推奨する。
- SSO採用時はIdP側ポリシーに従う。

### セッション管理

- 管理画面セッションはアイドルタイムアウト8時間を基本とする。
- Instagramアクセストークン期限は保存し、期限前通知を行う。
- 再認可後は旧トークンを即時失効扱いにする。

## データ保護

### 暗号化

- 転送中: TLS/SSL
- 保管時: AES-256

### データの分類

- 高機密: アクセストークン、Webhook URL、通知先資格情報
- 中機密: 投稿下書き、監査ログ、ジョブ実行結果
- 低機密: UI設定、画面表示用集計値

## アクセス制御

- RBACを採用する。
- 管理者: 全設定・全ログ閲覧・再実行可能
- 運用者: コンテンツ作成、予約、失敗確認、再実行可能
- 閲覧者: ダッシュボードとカレンダー閲覧のみ

## 脆弱性対策

- SQL インジェクション対策
- クロスサイトスクリプティング (XSS) 対策
- クロスサイトリクエストフォージェリ (CSRF) 対策
- 認可テスト

追加対策:
- ORMまたはプリペアドステートメントを利用する。
- キャプションなどの表示時はエスケープを徹底する。
- 管理画面の状態変更APIにはCSRF対策を適用する。
- ファイルアップロード時にMIMEタイプ・拡張子・サイズ検証を行う。
- シークレットはSecrets Manager等で管理し、平文保存しない。

## 監査ログ

- 連携、作成、編集、削除、予約、再実行、設定変更を監査対象とする。
- 監査ログには操作者、対象リソース、差分要約、時刻、requestId を含める。
- 保存期間は180日以上とする。

## 外部セキュリティレビュー

- リリース前にアプリケーションセキュリティレビューを実施する。
- 年1回以上の脆弱性診断を実施する。
- OAuth権限変更時は都度レビューする。
