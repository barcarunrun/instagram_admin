# ログとモニタリング

## ログ戦略

### ログレベル

- ERROR: エラー
- WARN: 警告
- INFO: 情報
- DEBUG: デバッグ

### ログ出力

- アプリケーションログはJSON構造化ログとして出力する。
- 出力先は集中ログ基盤（CloudWatch Logs, Datadog, ELKなど）とする。
- requestId, userId, contentId, jobId, accountId を相関キーとして記録する。

## メトリクス

### アプリケーションメトリクス

- リクエストレート
- レスポンスタイム
- エラーレート
- データベースクエリ数
- 投稿実行率
- 予約登録成功率
- ジョブ再試行回数
- 要再認可アカウント数

### インフラストラクチャメトリクス

- CPU使用率
- メモリ使用率
- ディスク使用率
- ネットワークトラフィック
- Redisキュー滞留数
- DB接続数

## アラート

- API 5xx エラー率が5分平均で3%超: 開発チャネルへ通知
- 投稿ジョブ失敗率が15分平均で10%超: 開発 + 運用チャネルへ通知
- 要再認可アカウント数が1件以上: 運用チャネルへ通知
- Workerキュー滞留時間が5分超: 開発チャネルへ通知
- DB接続枯渇率が80%超: SRE/開発チャネルへ通知

## ダッシュボード

- 経営/運用向け: 投稿実行率、週次投稿本数、失敗件数、要対応件数
- 開発向け: APIレイテンシ、ジョブ処理件数、再試行件数、外部APIエラー率
- インフラ向け: CPU、メモリ、Redis、DB、ネットワーク状況

## トラブルシューティング

1. requestId または jobId で対象リクエストを特定する。
2. posting_jobs の状態遷移と publish_results のレスポンスを確認する。
3. AUTH_EXPIRED, RATE_LIMITED, VALIDATION_ERROR などのエラー分類を確認する。
4. 同一時刻帯のInstagram Graph APIエラー率とRedisキュー滞留を確認する。
5. 必要に応じて監査ログから操作履歴を参照する。
