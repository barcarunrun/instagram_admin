# ロールバック計画

## ロールバック戦略

ロールバックが必要な場合は、以下の手順で実施します。

## ロールバック実行手順

### ステップ 1: 状況確認

```bash
# 現在のバージョン確認
npm run version

# エラーログ確認
tail -f logs/error.log
```

### ステップ 2: サービス停止

```bash
npm run stop
```

### ステップ 3: 前のバージョンに戻す

```bash
# 前のバージョンをデプロイ
npm run deploy:v0.9.0
```

### ステップ 4: データベースロールバック

```bash
# マイグレーション取り消し
npm run migrate:down

# バックアップから復元
npm run restore:backup
```

### ステップ 5: サービス再起動

```bash
npm run start
```

### ステップ 6: 動作確認

```bash
npm run health-check
```

## ロールバック時間

- 予想ロールバック時間: 15 分

## ロールバック後の対応

1. エラーログを分析
2. 問題を特定・修正
3. テスト環境で検証
4. 新しいバージョンを再度デプロイ
