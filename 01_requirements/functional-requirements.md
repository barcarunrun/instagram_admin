# 機能要件

## 機能要件一覧

### 機能 1: Instagramアカウント連携管理

**説明:**
Instagram Graph APIを利用するために、InstagramビジネスアカウントとFacebookページを連携し、投稿可能状態を維持する。

**入力:**
- Facebookログイン認可情報
- InstagramビジネスアカウントID
- FacebookページID

**出力:**
- 連携状態（未連携 / 有効 / 期限切れ / エラー）
- 投稿権限チェック結果

**処理フロー:**
1. OAuth認可を実行する。
2. 連携対象アカウントを選択する。
3. 必要権限を検証し、連携状態を保存する。
4. トークン期限を監視し、期限前に再認可を促す。

### 機能 2: 投稿コンテンツ管理（下書き・資産管理）

**説明:**
投稿に必要なメディア・キャプション・ハッシュタグ・リンク情報を下書きとして管理する。ブランド運用向けに再利用しやすい構造を提供する。

**入力:**
- メディアファイル（画像 / 動画）
- キャプション
- ハッシュタグ
- 投稿ラベル（商品カテゴリ、キャンペーン名など）

**出力:**
- 下書き投稿データ
- メディア検証結果（形式、サイズ、尺、アスペクト比）
- バージョン履歴

**処理フロー:**
1. ユーザーがメディアとテキストを登録する。
2. システムがAPI仕様に基づきメディアを検証する。
3. 問題がなければ下書き保存し、問題があれば修正ガイドを返す。
4. 保存済み下書きを検索・複製・更新できる。
5. 専用のメディア管理ページでアップロード済み資産を検索し、利用状況を確認できる。
6. 未使用メディアのみ削除可能とし、利用中資産は削除を拒否する。

### 機能 3: 投稿種別別の作成支援

**説明:**
Instagram APIで投稿可能な全コンテンツ種別に対応した作成UIを提供する。
対象はAPI仕様の更新に追従可能な構成とする。

**入力:**
- 投稿種別（画像フィード / 動画フィード / カルーセル / リール / その他APIで公開される投稿種別）
- 種別ごとの設定値（カバー画像、カルーセル順序、音源情報など）

**出力:**
- 種別に応じた投稿ペイロード
- 必須項目充足チェック結果

**処理フロー:**
1. ユーザーが投稿種別を選択する。
2. システムが種別ごとに必要な入力項目を表示する。
3. 必須項目と制約を検証する。
4. 投稿可能な標準形式へ正規化して保存する。

### 機能 4: 予約投稿管理

**説明:**
下書き投稿を日時指定で予約し、指定時刻に自動投稿する。運用の継続性を担保するため、キュー管理を行う。

**入力:**
- 投稿ID（下書きID）
- 公開予定日時（タイムゾーン付き）
- 公開先アカウント

**出力:**
- 予約ジョブ情報（予約済み / 実行中 / 成功 / 失敗 / 再試行中）
- 実行ログ

**処理フロー:**
1. ユーザーが公開日時を指定して予約する。
2. システムが重複時間帯・公開上限・過去日時を検証する。
3. 有効な予約をジョブキューへ登録する。
4. スケジューラが時刻到達時に投稿APIを呼び出す。
5. 結果を保存し、成功・失敗を通知する。

### 機能 5: 投稿実行・再試行・障害回復

**説明:**
APIエラーや一時障害時に、投稿失敗を最小化するための再試行と復旧オペレーションを提供する。

**入力:**
- 投稿ジョブID
- エラーコード
- 手動再実行指示

**出力:**
- 再試行結果
- 最終状態（成功 / 要対応）
- 障害理由（認証切れ、メディア不正、レート制限など）

**処理フロー:**
1. 投稿失敗時にエラー分類を行う。
2. 一時エラーは指数バックオフで自動再試行する。
3. 恒久エラーは要修正として運用者へ通知する。
4. 修正後に手動または自動で再実行する。

### 機能 6: 投稿カレンダーと運用可視化

**説明:**
投稿計画と実績を日次・週次で可視化し、投稿実行率と投稿頻度を管理できるようにする。

**入力:**
- 予約投稿データ
- 投稿実績データ
- 期間指定

**出力:**
- カレンダービュー（月 / 週）
- KPIビュー（投稿実行率、週次投稿本数）
- 未実行・失敗投稿一覧

**処理フロー:**
1. 予約情報と実績情報を統合する。
2. 期間単位で集計する。
3. KPIとの差分を表示する。
4. 未達が見込まれる場合にアラートを出す。

### 機能 7: 承認フロー（任意）

**説明:**
投稿前にレビュー・承認を挟める運用モードを提供し、ブランドトーンの品質を安定化する。

**入力:**
- 下書き投稿
- 承認者コメント
- 承認ステータス

**出力:**
- 承認履歴
- 差し戻し理由
- 承認済み予約可能状態

**処理フロー:**
1. 作成者が承認申請を行う。
2. 承認者が内容を確認する。
3. 承認または差し戻しを記録する。
4. 承認済みのみ予約可能にする。

### 機能 8: 監査ログ・運用通知

**説明:**
投稿に関する変更操作と実行結果を追跡し、運用事故時の調査と迅速復旧を可能にする。

**入力:**
- 操作イベント（作成、編集、削除、予約、投稿、再試行）
- 通知チャネル設定（メール、チャット）

**出力:**
- 監査ログ
- 失敗通知
- 日次サマリ通知

**処理フロー:**
1. 主要イベントを時系列で記録する。
2. 重要イベントを通知ルールに従い配信する。
3. 運用者がログ検索し、原因特定できる状態を維持する。
