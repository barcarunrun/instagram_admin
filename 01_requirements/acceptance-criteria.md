# 受入基準

## フォーマット

受入基準はGiven-When-Thenフォーマットで記述します。

## テストケース

### テストケース 1: Instagramアカウント連携成功

```
Given: InstagramビジネスアカウントとFacebookページが未連携である
When: 運用者がOAuth認可を完了し、対象アカウントを選択する
Then: 連携状態が「有効」となり、投稿権限チェックが成功する
```

### テストケース 2: コンテンツ下書き保存（画像フィード）

```
Given: 運用者が画像、キャプション、ハッシュタグを入力している
When: 下書き保存を実行する
Then: 下書きが保存され、メディア検証結果が「問題なし」で返る
```

### テストケース 3: コンテンツ下書き保存（動画フィード）

```
Given: 運用者が動画、キャプション、ハッシュタグを入力している
When: 下書き保存を実行する
Then: 下書きが保存され、動画仕様チェック結果が返る
```

### テストケース 4: コンテンツ下書き保存（カルーセル）

```
Given: 運用者が複数メディアをアップロードし、表示順を設定している
When: カルーセル下書きを保存する
Then: 順序情報を含む下書きが保存される
```

### テストケース 5: コンテンツ下書き保存（リール）

```
Given: 運用者がリール用動画と設定値を入力している
When: リール下書きを保存する
Then: リール種別として保存され、必須項目充足チェックが成功する
```

### テストケース 5-1: メディア管理ページで未使用メディアを削除できる

```
Given: 運用者がアップロード済みの未使用メディアを保有している
When: メディア管理ページから対象メディアを選択して削除する
Then: 対象メディアが一覧から消え、削除完了メッセージが表示される
```

### テストケース 5-2: 利用中メディアは削除できない

```
Given: 既存コンテンツで利用中のメディアが存在する
When: 運用者がメディア管理ページでそのメディアを選択する
Then: 削除操作が無効化される、または削除要求が拒否される
```

### テストケース 6: 予約投稿登録成功

```
Given: 承認済みの下書きがあり、未来時刻が指定されている
When: 運用者が予約投稿を登録する
Then: 予約ジョブが「予約済み」となり、カレンダーに表示される
```

### テストケース 7: 予約投稿自動実行成功

```
Given: 予約ジョブが「予約済み」で実行時刻に到達している
When: スケジューラが投稿ジョブを実行する
Then: 投稿が公開され、ジョブ状態が「成功」に更新される
```

### テストケース 8: 一時障害時の自動再試行

```
Given: 予約投稿実行時に一時的なAPI障害が発生している
When: システムがエラー分類を実行する
Then: ジョブが指数バックオフで自動再試行され、再試行履歴が記録される
```

### テストケース 9: 恒久エラー時の要対応化

```
Given: 投稿データ不備などの恒久エラーが発生している
When: システムがエラー分類を実行する
Then: ジョブ状態が「要対応」となり、運用者へ通知される
```

### テストケース 10: KPI可視化

```
Given: 計画投稿と実績投稿のデータが蓄積されている
When: 運用者が週次ダッシュボードを表示する
Then: 投稿実行率と週次投稿本数が表示され、未実行投稿が識別できる
```
