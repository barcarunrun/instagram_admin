# ユーザーストーリー

## ストーリー形式

ユーザーストーリーはBDD形式で記述します。

## ストーリー一覧

### ストーリー 1: アカウント連携

```
As a スモールビジネスの経営者
I want to Instagramアカウントを安全に連携したい
So that 投稿作業を自動化して毎日の運用負荷を下げたい
```

**受入基準:**
- [ ] 連携完了後に投稿権限チェックが成功する
- [ ] トークン期限切れ前に再認可を促される

### ストーリー 2: コンテンツ下書き管理

```
As a SNS運用担当者
I want to 投稿コンテンツを下書きとして管理したい
So that 投稿品質を保ちながら短時間で公開準備したい
```

**受入基準:**
- [ ] 画像・動画を含む下書きを保存、編集、複製できる
- [ ] 保存時にメディア仕様チェック結果が表示される

### ストーリー 3: 投稿種別対応

```
As a SNS運用担当者
I want to APIで投稿可能な全コンテンツ種別を選んで投稿を作成したい
So that 施策に応じて最適なフォーマットを使い分けたい
```

**受入基準:**
- [ ] 画像フィード、動画フィード、カルーセル、リールを作成できる
- [ ] APIで新種別が追加された際に拡張可能な構造になっている

### ストーリー 4: 予約投稿

```
As a スモールビジネスの経営者
I want to 投稿を日時指定で予約したい
So that 毎日手動で投稿しなくても継続発信したい
```

**受入基準:**
- [ ] 未来時刻で予約登録できる
- [ ] 指定時刻に自動投稿され、結果が記録される

### ストーリー 5: 失敗時の復旧

```
As a SNS運用担当者
I want to 投稿失敗時に再試行と原因確認をしたい
So that 投稿実行率を高く維持したい
```

**受入基準:**
- [ ] 一時障害は自動再試行される
- [ ] 恒久エラーは要対応として通知される

### ストーリー 6: 運用可視化

```
As a 事業責任者
I want to 投稿計画と実績を可視化したい
So that 投稿実行率や頻度の未達を早期に把握したい
```

**受入基準:**
- [ ] カレンダーで予約と実績を確認できる
- [ ] 週次投稿本数と投稿実行率が表示される
